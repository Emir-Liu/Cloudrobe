# 项目管理参考资料

## 项目管理最佳实践

### 敏捷开发原则

1. **客户满意优先**
   - 尽早交付有价值的软件
   - 频繁交付（2-4 周一个迭代）

2. **拥抱变化**
   - 需求变更应及时响应
   - 但要评估变更影响

3. **持续交付**
   - 保持可部署状态
   - 自动化测试和部署

4. **团队协作**
   - 业务和开发紧密协作
   - 每日站会保持同步

### 项目管理三约束

```
        时间
         /\
        /  \
       /    \
      /      \
     /________\
   成本  范围
```

- **时间**: 项目进度
- **成本**: 资源投入
- **范围**: 功能需求

**约束关系**: 调整任何一个约束都会影响其他两个

### 风险矩阵

| 影响 \ 概率 | 低 | 中 | 高 |
|------------|---|---|---|
| 高 | 🟡 监控 | 🟠 降低 | 🔴 规避 |
| 中 | 🟢 接受 | 🟡 监控 | 🟠 降低 |
| 低 | 🟢 接受 | 🟢 接受 | 🟡 监控 |

## LifeManager 项目特定参考

### Phase 1 项目时间线

```
2026-01-01  项目启动
2026-01-05  技术选型完成
2026-01-10  架构设计完成
2026-01-15  后端开发完成
2026-01-20  前端开发完成
2026-01-25  联调测试完成
2026-01-28  MVP 上线
```

### 里程碑验收标准

| 里程碑 | 验收标准 | 验收人 |
|--------|---------|--------|
| 技术选型 | 技术架构文档完成,技术栈确定 | 技术架构师 |
| 产品设计 | 产品需求文档、UI设计文档完成 | 产品经理 |
| 后端开发 | 所有 API 接口开发完成,测试通过 | 后端开发工程师 |
| 前端开发 | 所有页面开发完成,可联调 | 前端开发工程师 |
| MVP 上线 | 核心功能可用,通过验收测试 | 项目经理 |

### 团队协作规则

1. **代码提交规范**
   - 每个提交只包含一个功能
   - 提交信息清晰: "feat: 添加用户登录功能"
   - 提交前先拉取最新代码

2. **代码审查流程**
   - PR 必须至少 1 人审查
   - 审查重点: 代码质量、测试覆盖、文档更新
   - 审查通过后才能合并

3. **文档更新原则**
   - 功能变更必须更新相关文档
   - 技术决策必须记录在文档
   - 每周同步文档更新情况

## 常见项目管理问题及解决方案

### 问题1: 项目延期

**症状**: 任务完成时间超过预期

**原因分析**:
- 任务估时不准
- 需求变更频繁
- 技术难度超预期

**解决方案**:
- 预留 20% 缓冲时间
- 加强需求评审,控制变更
- 提前进行技术验证

### 问题2: 质量问题

**症状**: Bug 数量多,测试不充分

**原因分析**:
- 测试覆盖不足
- 代码审查不严格
- 缺少自动化测试

**解决方案**:
- 提高测试覆盖率要求 (>80%)
- 强化代码审查流程
- 引入自动化测试

### 问题3: 沟通不畅

**症状**: 信息传递不及时,理解不一致

**原因分析**:
- 缺少有效沟通渠道
- 文档更新不及时
- 需求传递失真

**解决方案**:
- 建立每日站会
- 所有决策记录在文档
- 使用 @ 提及相关人员

## 项目管理工具

### 任务管理

使用 `todo_write` 工具管理任务:

```python
# 创建新任务列表
todo_write(merge=False, todos=[
    {"id": "1", "status": "pending", "content": "用户注册功能开发"},
    {"id": "2", "status": "pending", "content": "登录功能开发"}
])

# 更新任务状态
todo_write(merge=True, todos=[
    {"id": "1", "status": "in_progress", "content": "用户注册功能开发"}
])

# 完成任务
todo_write(merge=True, todos=[
    {"id": "1", "status": "completed", "content": "用户注册功能开发"}
])
```

### 技能调用

根据任务类型调用相关技能:

| 任务类型 | 调用技能 |
|---------|---------|
| 产品需求分析 | `use_skill(产品经理)` |
| 技术架构设计 | `use_skill(技术架构师)` |
| 后端开发 | `use_skill(python后端开发工程师)` |
| 前端开发 | `use_skill(vue3前端开发工程师)` |
| UI/UX 设计 | `use_skill(ui/ux设计师)` |
| CI/CD 部署 | `use_skill(devops工程师)` |
| 项目管理 | `use_skill(项目经理)` |
