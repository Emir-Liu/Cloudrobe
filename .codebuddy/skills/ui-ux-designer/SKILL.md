# UI/UX设计师技能

## 概述

本技能提供专业的UI/UX设计能力,涵盖从用户研究到高保真设计的全流程。帮助创建美观、易用、符合用户体验标准的界面设计。

## 核心能力

### 1. 用户研究(User Research)

#### 用户画像(Persona)
创建详细的用户画像,包含:
- **基本信息**: 姓名、年龄、职业、教育背景
- **行为特征**: 使用习惯、偏好、痛点
- **目标与动机**: 使用产品的核心目标
- **场景描述**: 典型使用场景和情境

用户画像模板:
```
### [角色名称]

**基本信息**
- 年龄: XX岁
- 职业: XXX
- 技术水平: [入门/中级/高级]

**行为特征**
- 使用场景: 描述具体使用场景
- 痛点: 用户面临的问题
- 偏好: 用户的喜好和习惯

**目标与动机**
- 主要目标: 用户想要达成什么
- 次要目标: 辅助性目标
- 动机: 为什么使用产品

**场景描述**
```

#### 用户旅程图(User Journey Map)
绘制用户从发现到使用的完整旅程:
- 触发阶段(Touchpoints)
- 用户行为(Actions)
- 情感曲线(Emotion)
- 痛点(Pain Points)
- 机会点(Opportunities)

### 2. 信息架构(Information Architecture)

#### 站点地图(Sitemap)
创建清晰的站点地图,展示:
- 主要页面和子页面
- 页面之间的层级关系
- 用户导航路径
- 内容分类结构

```
首页
├── 关于我们
│   ├── 公司简介
│   └── 团队介绍
├── 产品服务
│   ├── 产品A
│   ├── 产品B
│   └── 定制服务
├── 资源中心
│   ├── 博客
│   ├── 文档
│   └── 视频教程
└── 联系我们
    ├── 在线客服
    └── 意见反馈
```

#### 内容模型(Content Model)
定义内容类型和字段:
- 内容类型(Content Types)
- 字段定义(Fields)
- 关系(Relationships)
- 展示规则(Display Rules)

### 3. 线框图设计(Wireframe)

#### 线框图层级
- **低保真线框图**: 快速草图,重点在布局和结构
- **中保真线框图**: 添加交互和流程
- **高保真线框图**: 接近最终设计,包含视觉细节

#### 线框图设计原则
- **内容优先**: 内容驱动设计,而非装饰
- **层次清晰**: 使用大小、间距、对比建立视觉层次
- **一致性强**: 保持组件和模式的一致性
- **响应式设计**: 考虑不同设备的适配

### 5. 交互设计(Interaction Design)

#### 微交互(Micro-interactions)
- 按钮悬停和点击效果
- 加载动画
- 过渡动画
- 表单验证反馈
- 成功/失败提示

#### 交互动效
- **进入动画**: Fade in, Slide in, Scale up
- **退出动画**: Fade out, Slide out, Scale down
- **切换动画**: Tab切换, 页面转场
- **滚动动画**: 列表加载, 无限滚动

#### 交互模式
- **导航模式**: 顶部导航、侧边导航、抽屉式导航
- **数据输入模式**: 表单、向导、批量操作
- **数据展示模式**: 列表、卡片、图表、日历
- **操作模式**: 按钮、操作栏、下拉菜单

### 6. 响应式设计(Responsive Design)

#### 断点设置(Breakpoints)
```
移动设备:
- 小屏手机: < 576px
- 大屏手机: 576px - 768px

平板设备:
- 平板: 768px - 992px

桌面设备:
- 小屏桌面: 992px - 1200px
- 大屏桌面: > 1200px
```

#### 布局策略
- **移动优先(Mobile First)**: 从小屏开始设计,向上适配
- **内容优先(Content First)**: 核心内容在所有设备上优先展示
- **渐进增强(Progressive Enhancement)**: 基础功能在所有设备可用,高级功能在大屏增强

### 9. 可访问性设计(Accessibility)

#### WCAG 2.1 指南
- **感知性(Perceivable)**: 内容可被所有用户感知
- **可操作性(Operable)**: 所有用户可操作界面
- **可理解性(Understandable)**: 信息清晰易懂
- **健壮性(Robust)**: 兼容辅助技术

#### 实践要点
- 色彩对比度至少4.5:1(文本)
- 提供键盘导航支持
- 图片添加alt文本
- 表单提供清晰的标签
- 错误信息明确提示
- 支持屏幕阅读器

## 设计流程

### 设计流程标准步骤

1. **需求分析**
   - 理解产品目标和用户需求
   - 分析竞品和行业标准
   - 确定设计范围和优先级

2. **用户研究**
   - 创建用户画像
   - 分析用户场景
   - 绘制用户旅程图

3. **信息架构**
   - 设计站点地图
   - 定义内容模型
   - 规划导航结构

4. **线框图设计**
   - 绘制低保真线框图
   - 进行内部评审
   - 迭代优化方案

5. **视觉设计**
   - 建立设计系统
   - 设计高保真界面
   - 创建组件库

6. **交互设计**
   - 设计交互流程
   - 添加动效
   - 优化用户体验

7. **原型制作**
   - 制作可交互原型
   - 进行用户测试
   - 收集反馈迭代

8. **设计交付**
   - 整理设计稿和标注
   - 编写设计文档
   - 与开发团队对接

## 设计趋势

### 当前设计趋势
- **极简主义**: 简洁设计,突出核心
- **暗黑模式**: 护眼节能,个性化选择
- **3D设计**: 增加视觉冲击力
- **微动效**: 提升用户体验
- **玻璃拟态**: 透明模糊效果
- **渐变设计**: 增加视觉层次
- **插图设计**: 个性化表达
- **深色背景**: 高级感设计
