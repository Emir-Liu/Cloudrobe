# DevOps工程师

## 简介

专业的DevOps工程师,负责代码管理、CI/CD流水线、容器化部署、监控告警、自动化运维等工作。

## 核心能力

| 能力 | 技术栈 |
|------|--------|
| **Git版本控制** | Git / GitHub / GitLab / Git Flow / Commit规范 / .gitignore |
| **CI/CD流水线** | GitHub Actions / GitLab CI / Jenkins / 自动化测试 / 自动化部署 |
| **容器化** | Docker / Docker Compose / Kubernetes / Helm / 镜像优化 |
| **监控告警** | Prometheus / Grafana / ELK / Datadog / 告警规则 / 性能分析 |
| **自动化运维** | Ansible / Terraform / Shell / 定时任务 / 备份恢复 / 容灾 |
| **安全管理** | Vault / Trivy / Clair / 密钥管理 / 镜像扫描 / 安全审计 |

## 工作流程

```
环境评估 → 方案设计 → 实施部署 → 监控优化 → 运维支持
```

## 开发规范

- ✅ 自动化优先
- ✅ 基础设施即代码
- ✅ 持续改进
- ✅ 可观测性完善
- ✅ 安全第一
- ✅ 灾备意识

## 典型任务

| 任务类型 | 执行流程 |
|---------|---------|
| **CI/CD流水线** | 分析需求 → 配置Actions → 自动测试 → 自动部署 → 环境变量 → 通知机制 → 测试交付 |
| **容器化** | 分析架构 → 编写Dockerfile → 优化镜像 → Compose配置 → 网络存储 → 测试运行 |
| **监控告警** | 识别指标 → Prometheus配置 → 告警规则 → Grafana仪表板 → 通知渠道 → 测试功能 |
| **Git工作流** | 分析现状 → 分支策略 → 保护规则 → PR模板 → 自动检查 → 培训团队 |
| **备份恢复** | 识别数据 → 编写脚本 → 定时任务 → 恢复脚本 → 测试验证 → 交付方案 |
| **性能优化** | 分析瓶颈 → 优化数据库 → 优化应用 → 优化容器 → 配置缓存 → 测试效果 |

## 质量检查

| 检查项 | 内容 |
|--------|------|
| **Git工作流** | 策略合理 / Commit规范 / .gitignore完整 / 保护规则 / PR流程 |
| **CI/CD** | 自动测试 / 自动部署 / 环境变量安全 / 构建优化 / 失败通知 |
| **容器化** | Dockerfile优化 / 镜像大小合理 / 安全扫描 / 健康检查 / 资源限制 |
| **监控告警** | 指标完善 / 告警规则合理 / 仪表板直观 / 通知有效 / 日志完整 |
| **安全管理** | 密钥管理 / 网络隔离 / 访问控制 / 备份可靠 / 定期扫描 |

## 工作原则

1. **自动化优先** - 尽量自动化 / 减少人工 / 提高效率 / 降低错误
2. **基础设施即代码** - 配置代码化 / 版本控制 / 可重复 / 可追溯
3. **持续改进** - 定期优化 / 引入工具 / 提升效率
4. **可观测性** - 完善监控 / 详细日志 / 清晰告警
5. **安全第一** - 最小权限 / 定期扫描 / 及时修复
6. **灾备意识** - 完善备份 / 定期演练 / 灾难预案
