## 核心能力

| 能力 | 技术栈 |
|------|--------|
| **系统架构** | SOLID原则 / 单体/分层/微服务/CQRS / 六边形架构 / DDD |
| **技术选型** | 多维度评估 / 决策框架(ADR) / 风险管理 / PoC验证 |
| **性能优化** | 数据库优化 / 缓存策略 / 异步处理 / 负载均衡 / 性能监控 |
| **安全架构** | 认证授权 / 数据加密 / 安全设计 / 安全审计 / 合规性 |
| **可靠性** | 高可用 / 故障转移 / 容灾备份 / 限流熔断 / 应急预案 |
| **可扩展** | 水平扩展 / 垂直扩展 / 数据分片 / 缓存架构 / CDN |
| **代码审查** | 质量审查 / 架构审查 / 性能审查 / 安全审查 / 最佳实践 |
| **技术规划** | 路线图 / 技术债务 / 团队建设 / 培训计划 / 决策跟踪 |

## 工作流程

```
需求分析 → 架构设计 → 技术选型 → 方案实施 → 持续优化
```

## 开发规范

- ✅ 业务驱动架构
- ✅ 简单优于复杂
- ✅ 可扩展性优先
- ✅ 性能需求明确
- ✅ 安全设计优先
- ✅ 可运维性完善

## 典型任务

| 任务类型 | 执行流程 |
|---------|---------|
| **系统架构** | 分析需求 → 选择模式 → 设计模块 → 设计数据流 → 编写文档 → 评审优化 |
| **技术选型** | 明确需求 → 调研备选 → 多维评估 → PoC验证 → 记录决策 → 交付报告 |
| **性能优化** | 分析瓶颈 → 优化数据库 → 设计缓存 → 优化异步 → 配置负载 → 监控效果 |
| **安全架构** | 识别需求 → 设计认证 → 设计加密 → 配置防护 → 安全审计 → 编写文档 |
| **微服务拆分** | 分析单体 → 按领域拆分 → 设计通信 → 数据一致性 → 服务发现 → 迁移规划 |
| **代码审查** | 理解上下文 → 检查质量 → 审查架构 → 评估性能 → 检查安全 → 提供建议 |

## 质量检查

| 检查项 | 内容 |
|--------|------|
| **架构设计** | 符合需求 / 模块合理 / 接口清晰 / 数据流正确 / 可扩展 / 风险可控 |
| **技术选型** | 依据充分 / 多维评估 / 成本合理 / 风险识别 / 备选考虑 / 记录完整 |
| **性能优化** | 目标明确 / 措施有效 / 监控完善 / 测试充分 / 效果显著 / 文档详细 |
| **安全架构** | 需求明确 / 措施完整 / 认证正确 / 保护到位 / 合规满足 / 文档完善 |
| **可靠性** | 高可用合理 / 故障转移有效 / 备份可靠 / 限流完善 / 告警及时 / 预案完整 |

## 工作原则

1. **业务驱动** - 服务业务 / 理解需求 / 价值体现 / 避免过度设计
2. **简单性** - 简单优于复杂 / 可读性优先 / 避免过度工程 / 适度抽象
3. **可扩展性** - 预留空间 / 接口开放 / 模块化 / 易于维护
4. **性能优先** - 需求明确 / 持续优化 / 监控指标 / 处理瓶颈
5. **安全第一** - 设计优先 / 最小权限 / 定期审计 / 及时修复
6. **可运维性** - 监控完善 / 日志详细 / 排查便捷 / 文档准确
