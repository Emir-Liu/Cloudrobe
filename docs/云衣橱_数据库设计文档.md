## 2. 数据库选型

### 2.1 主数据库: PostgreSQL 15+
**选择理由**:
- ACID事务支持，保证数据一致性
- JSONB字段，灵活存储复杂数据
- 全文搜索能力，支持商品搜索
- 成熟稳定，社区活跃
- 支持复杂查询和聚合

### 2.2 缓存数据库: Redis 7+
**选择理由**:
- 高性能读写
- 丰富的数据结构
- 支持持久化
- 可作为消息队列

## 4. 表结构详细设计

### 4.1 users (用户表)

#### 表结构
```sql
CREATE TABLE users (
    -- 主键
    id BIGSERIAL PRIMARY KEY,
    
    -- 账号信息
    openid VARCHAR(128) UNIQUE,           -- 微信openid
    phone VARCHAR(20) UNIQUE,            -- 手机号
    
    -- 基本信息
    nickname VARCHAR(50),                -- 昵称
    avatar VARCHAR(500),                 -- 头像URL
    gender SMALLINT DEFAULT 0,            -- 性别 0未知 1男 2女
    height SMALLINT,                      -- 身高cm
    weight SMALLINT,                      -- 体重kg
    size_preferences JSONB,               -- 尺码偏好 {"top":"M","bottom":"28"}
    bio TEXT,                            -- 个人简介
    
    -- 信用与资金
    credit_score INTEGER DEFAULT 0,       -- 信用积分
    credit_level VARCHAR(20),             -- 信用等级
    balance DECIMAL(10,2) DEFAULT 0,     -- 余额（元）
    
    -- 实名认证
    is_verified BOOLEAN DEFAULT FALSE,    -- 是否实名认证
    id_card_name VARCHAR(50),            -- 身份证姓名
    id_card_number VARCHAR(18),          -- 身份证号
    id_card_front VARCHAR(500),          -- 身份证正面照片URL
    id_card_back VARCHAR(500),           -- 身份证背面照片URL
    verified_at TIMESTAMP,               -- 认证时间
    
    -- 状态
    status SMALLINT DEFAULT 1,           -- 状态 1正常 2冻结
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- 索引
    INDEX idx_phone (phone),
    INDEX idx_openid (openid),
    INDEX idx_credit_score (credit_score),
    INDEX idx_status (status)
);
```

### 4.2 clothings (衣物表)

#### 表结构
```sql
CREATE TABLE clothings (
    -- 主键
    id BIGSERIAL PRIMARY KEY,
    
    -- 所属信息
    owner_id BIGINT NOT NULL,           -- 出租方ID
    name VARCHAR(100) NOT NULL,         -- 衣物名称
    
    -- 基本信息
    brand VARCHAR(50),                  -- 品牌
    category VARCHAR(50) NOT NULL,      -- 分类 上装/下装/裙装/礼服/外套
    size VARCHAR(10) NOT NULL,          -- 尺码 XS/S/M/L/XL/XXL
    condition VARCHAR(20) NOT NULL,     -- 新旧程度 全新/95新/9成新/8成新/85新
    description TEXT,                   -- 描述
    images JSONB NOT NULL,             -- 图片URL数组 ["url1","url2","url3"]
    
    -- 租赁信息
    daily_rent DECIMAL(10,2) NOT NULL,  -- 日租金（元）
    deposit DECIMAL(10,2) NOT NULL,     -- 押金（元）
    min_rent_days SMALLINT DEFAULT 1,   -- 最短租期（天）
    max_rent_days SMALLINT DEFAULT 7,   -- 最长租期（天）
    require_wash BOOLEAN DEFAULT TRUE,-- 是否需要清洗
    
    -- 配送信息
    delivery_type SMALLINT DEFAULT 1,  -- 配送方式 1快递到付 2顺丰包邮 3自提
    delivery_fee DECIMAL(10,2) DEFAULT 0, -- 运费（元）
    
    -- 状态
    status SMALLINT DEFAULT 1,         -- 1可租 2租赁中 3已下架
    
    -- 统计数据
    rent_count INTEGER DEFAULT 0,       -- 租赁次数
    total_revenue DECIMAL(10,2) DEFAULT 0, -- 总收益（元）
    rating_avg DECIMAL(3,2),           -- 平均评分
    rating_count INTEGER DEFAULT 0,    -- 评分数量
    
    -- 时间
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- 索引
    INDEX idx_owner (owner_id),
    INDEX idx_category (category),
    INDEX idx_size (size),
    INDEX idx_status (status),
    INDEX idx_price (daily_rent),
    INDEX idx_created (created_at DESC),
    FULLTEXT idx_search (name, description),
    
    -- 外键约束
    FOREIGN KEY (owner_id) REFERENCES users(id)
);
```

#### 字段说明
| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| id | BIGSERIAL | 主键ID | 是 |
| owner_id | BIGINT | 出租方ID | 是 |
| name | VARCHAR(100) | 衣物名称 | 是 |
| brand | VARCHAR(50) | 品牌 | 否 |
| category | VARCHAR(50) | 分类 | 是 |
| size | VARCHAR(10) | 尺码 | 是 |
| condition | VARCHAR(20) | 新旧程度 | 是 |
| description | TEXT | 描述 | 否 |
| images | JSONB | 图片URL数组 | 是 |
| daily_rent | DECIMAL(10,2) | 日租金（元） | 是 |
| deposit | DECIMAL(10,2) | 押金（元） | 是 |
| min_rent_days | SMALLINT | 最短租期（天） | 否 |
| max_rent_days | SMALLINT | 最长租期（天） | 否 |
| require_wash | BOOLEAN | 是否需要清洗 | 否 |

### 4.3 orders (订单表)

#### 表结构
```sql
CREATE TABLE orders (
    -- 主键
    id BIGSERIAL PRIMARY KEY,
    order_no VARCHAR(32) UNIQUE NOT NULL,  -- 订单号
    
    -- 关联信息
    clothing_id BIGINT NOT NULL,          -- 衣物ID
    renter_id BIGINT NOT NULL,            -- 租赁方ID
    owner_id BIGINT NOT NULL,             -- 出租方ID
    
    -- 租赁信息
    start_date DATE NOT NULL,             -- 开始日期
    end_date DATE NOT NULL,               -- 结束日期
    rent_days SMALLINT NOT NULL,          -- 租赁天数
    
    -- 金额信息
    daily_rent DECIMAL(10,2) NOT NULL,    -- 日租金
    rent_amount DECIMAL(10,2) NOT NULL,   -- 租金总额（元）
    deposit DECIMAL(10,2) NOT NULL,       -- 押金（元）
    delivery_fee DECIMAL(10,2) DEFAULT 0,-- 运费（元）
    total_amount DECIMAL(10,2) NOT NULL,  -- 订单总额（元）
    
    -- 订单状态流转
    status SMALLINT NOT NULL,             -- 1待确认 2待发货 3待收货 4租赁中 5待归还 6已完成 7已取消 8售后中
    confirm_time TIMESTAMP,               -- 确认时间
    ship_time TIMESTAMP,                  -- 发货时间
    receive_time TIMESTAMP,              -- 收货时间
    return_time TIMESTAMP,                -- 归还时间
    complete_time TIMESTAMP,             -- 完成时间
    
    -- 物流信息
    express_company VARCHAR(50),         -- 快递公司
    express_no VARCHAR(50),             -- 快递单号
    
    -- 评价
    renter_rating SMALLINT,              -- 租赁方评分 1-5
    renter_comment TEXT,                 -- 租赁方评价
    renter_images JSONB,                -- 租赁方评价图片
    owner_rating SMALLINT,               -- 出租方评分 1-5
    owner_comment TEXT,                  -- 出租方评价
    owner_images JSONB,                 -- 出租方评价图片
    
    -- 售后
    dispute_reason VARCHAR(100),         -- 争议原因
    dispute_desc TEXT,                   -- 争议描述
    dispute_images JSONB,                -- 争议图片
    dispute_status SMALLINT DEFAULT 0,   -- 争议状态 0无争议 1处理中 2已解决
    dispute_result TEXT,                 -- 争议结果
    deposit_refund DECIMAL(10,2),        -- 实际退还押金（元）
    
    -- 时间
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- 索引
    INDEX idx_order_no (order_no),
    INDEX idx_renter (renter_id),
    INDEX idx_owner (owner_id),
    INDEX idx_clothing (clothing_id),
    INDEX idx_status (status),
    INDEX idx_created (created_at DESC),
    INDEX idx_renter_status (renter_id, status),
    
    -- 外键约束
    FOREIGN KEY (clothing_id) REFERENCES clothings(id),
    FOREIGN KEY (renter_id) REFERENCES users(id),
    FOREIGN KEY (owner_id) REFERENCES users(id)
);
```

#### 字段说明
| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| id | BIGSERIAL | 主键ID | 是 |
| order_no | VARCHAR(32) | 订单号，唯一 | 是 |
| clothing_id | BIGINT | 衣物ID | 是 |
| renter_id | BIGINT | 租赁方ID | 是 |
| owner_id | BIGINT | 出租方ID | 是 |
| start_date | DATE | 开始日期 | 是 |
| end_date | DATE | 结束日期 | 是 |
| rent_days | SMALLINT | 租赁天数 | 是 |
| daily_rent | DECIMAL(10,2) | 日租金 | 是 |
| rent_amount | DECIMAL(10,2) | 租金总额 | 是 |
| deposit | DECIMAL(10,2) | 押金 | 是 |
| delivery_fee | DECIMAL(10,2) | 运费 | 否 |
| total_amount | DECIMAL(10,2) | 订单总额 | 是 |
| status | SMALLINT | 订单状态 | 是 |

#### 订单状态流转
```
1待确认 → 2待发货 → 3待收货 → 4租赁中 → 5待归还 → 6已完成
              ↓           ↓                              ↓
            7已取消     8售后中 ←────────────────────────┘
```

### 4.4 reviews (评价表)

#### 表结构
```sql
CREATE TABLE reviews (
    id BIGSERIAL PRIMARY KEY,
    order_id BIGINT NOT NULL,           -- 订单ID
    reviewer_id BIGINT NOT NULL,        -- 评价人ID
    target_id BIGINT NOT NULL,          -- 被评价人ID
    type SMALLINT NOT NULL,             -- 1租赁方评价出租方 2出租方评价租赁方
    rating SMALLINT NOT NULL,           -- 评分 1-5
    comment TEXT,                       -- 评价内容
    images JSONB,                      -- 评价图片URL数组
    created_at TIMESTAMP DEFAULT NOW(),
    
    -- 索引
    INDEX idx_reviewer (reviewer_id),
    INDEX idx_target (target_id),
    INDEX idx_order (order_id),
    
    -- 外键约束
    FOREIGN KEY (order_id) REFERENCES orders(id),
    FOREIGN KEY (reviewer_id) REFERENCES users(id),
    FOREIGN KEY (target_id) REFERENCES users(id)
);
```

### 4.5 favorites (收藏表)

```sql
CREATE TABLE favorites (
    id BIGSERIAL PRIMARY KEY,
```

### 4.6 messages (消息表)

#### 表结构
```sql
CREATE TABLE messages (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,            -- 接收用户ID
    type SMALLINT NOT NULL,             -- 消息类型 1订单通知 2系统通知
    title VARCHAR(100) NOT NULL,        -- 消息标题
```

### 4.7 transactions (交易记录表)

#### 表结构
```sql
CREATE TABLE transactions (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,            -- 用户ID
    order_id BIGINT,                    -- 订单ID（可选）
    type SMALLINT NOT NULL,             -- 类型 1租金收入 2押金收入 3租金支出 4押金支出 5退款 6提现
    amount DECIMAL(10,2) NOT NULL,      -- 金额
    balance_before DECIMAL(10,2),       -- 操作前余额
    balance_after DECIMAL(10,2),        -- 操作后余额
    description VARCHAR(200),           -- 描述
    created_at TIMESTAMP DEFAULT NOW(),
    
    -- 索引
    INDEX idx_user (user_id),
    INDEX idx_order (order_id),
    INDEX idx_type (type),
    INDEX idx_created (created_at DESC),
    
    -- 外键约束
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (order_id) REFERENCES orders(id)
);
```

## 7. 性能优化建议

### 7.1 索引优化
- 所有外键字段创建索引
- 常用查询条件创建索引

### 7.2 查询优化
- 避免SELECT *
- 使用LIMIT分页
- 避免N+1查询
- 使用EXPLAIN分析查询
