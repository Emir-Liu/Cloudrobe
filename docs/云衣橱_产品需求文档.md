## 2. 项目概述

### 2.1 项目背景
随着共享经济的发展和环保意识的提升，服装租赁市场逐渐兴起。用户希望：
- **提高衣物利用率**：闲置衣物获得新价值
- **降低时尚消费成本**：以较低价格体验高端品牌
- **减少资源浪费**：延长服装生命周期

### 2.2 产品定位
**Cloudrobe 云衣橱** - 基于P2P模式的服装租赁共享平台，连接衣橱主人与租赁者，实现闲置衣物的价值最大化。

### 2.3 核心价值
- **对出租方**：闲置衣物变现，减少浪费，环保贡献
- **对租赁方**：低门槛体验高端品牌，降低时尚成本
- **对平台**：连接供需，提升服装利用率

## 3. 用户画像

### 3.1 出租方（衣橱主人）
- **基本信息**：25-40岁女性为主，注重时尚，拥有较多衣物
- **核心诉求**：闲置衣物变现、环保理念、分享欲望
- **使用场景**：
  - 季节更替，部分衣物暂时不需要
  - 体重变化导致不合身的衣物
  - 活动服装（礼服、演出服）使用频率低

### 3.2 租赁方（时尚爱好者）
- **基本信息**：18-35岁，追求时尚但预算有限
- **核心诉求**：低成本试错、体验不同风格、参加特殊场合
- **使用场景**：
  - 参加婚礼、年会等正式场合需要礼服
  - 想要尝试新风格但不想购买
  - 旅行拍照需要多样化的服装

## 4. 功能需求

### 4.1 MVP 核心功能（第一阶段）

#### 4.1.1 用户认证
**用户故事**:
```
作为新用户,
我希望通过手机号注册/登录,
以便开始使用云衣橱服务
```

**验收标准**:
- [ ] 支持手机号+验证码注册/登录
- [ ] 支持微信一键登录
- [ ] 实名认证（上传身份证照片）

---

#### 4.1.2 衣物发布
**用户故事**:
```
作为出租方,
我希望发布自己的衣物信息,
以便让别人租借
```

**验收标准**:
- [ ] 填写衣物信息（品牌、尺码、新旧程度、押金、租金）
- [ ] 上传3-5张清晰照片（正面、背面、细节）
- [ ] 设置租赁规则（最短租期、最长租期、是否可清洗）
- [ ] 设置配送方式（自提/快递）
- [ ] 管理已发布的衣物（下架、编辑、删除）

---

#### 4.1.3 衣物浏览与搜索
**用户故事**:
```
作为租赁方,
我希望浏览和搜索可租借的衣物,
以便找到合适的衣服
```

**验收标准**:
- [ ] 首页推荐热门/最新衣物
- [ ] 按分类筛选（上装/下装/裙装/礼服/外套等）
- [ ] 按尺码筛选（XS/S/M/L/XL/XXL）
- [ ] 按品牌筛选
- [ ] 按价格区间筛选
- [ ] 关键词搜索
- [ ] 查看衣物详情（照片、描述、评价、出租方信息）

---

#### 4.1.4 租赁下单
**用户故事**:
```
作为租赁方,
我希望下单租借衣物,
以便拿到衣服
```

**验收标准**:
- [ ] 选择租赁日期（起止日期，最短1天）
- [ ] 查看租金和押金明细
- [ ] 支付租金和押金（微信支付/支付宝）
- [ ] 下单成功后通知出租方
- [ ] 查看订单状态（待确认、待发货、租赁中、待归还、已完成）

---

#### 4.1.5 订单管理
**用户故事**:
```
作为用户,
我希望查看和管理我的订单,
以便了解租赁状态
```

**验收标准**:
- [ ] 出租方：查看租借申请，确认或拒绝订单
- [ ] 出租方：发货，填写物流信息
- [ ] 租赁方：确认收货
- [ ] 租赁方：归还衣物
- [ ] 出租方：确认收到，退回押金
- [ ] 双方都可以查看订单进度

---

#### 4.1.6 评价系统
**用户故事**:
```
作为用户,
我希望评价租赁体验,
以便为其他用户参考
```

**验收标准**:
- [ ] 租赁方评价出租方（衣物描述准确度、沟通顺畅度）
- [ ] 出租方评价租赁方（爱护程度、归还及时性）
- [ ] 5星评分+文字评价
- [ ] 查看历史评价

---

#### 4.1.7 消息通知
**用户故事**:
```
作为用户,
我希望及时接收订单相关通知,
以便快速响应
```

**验收标准**:
- [ ] 新订单提醒
- [ ] 订单状态变更提醒
- [ ] 归还提醒（到期前1天）
- [ ] 押金退还提醒

---

### 4.2 第二阶段增强功能

#### 4.2.1 智能推荐
**用户故事**:
```
作为租赁方,
我希望系统推荐符合我风格的衣物,
以便更快找到喜欢的
```

**验收标准**:
- [ ] 基于历史浏览记录推荐
- [ ] 基于身材尺码推荐
- [ ] 基于用户标签偏好推荐

#### 4.2.2 信用体系
**用户故事**:
```
作为用户,
我希望建立自己的信用等级,
以便获得更好的交易体验
```

**验收标准**:
- [ ] 信用积分系统（按时归还+1分，违约-5分）
- [ ] 信用等级展示（普通/良好/优秀/信用之星）
- [ ] 高信用用户：押金减免、优先推荐
- [ ] 低信用用户：限制租赁、提高押金

#### 4.2.3 争议处理
**用户故事**:
```
作为用户,
我希望遇到问题能申请平台介入,
以便快速解决纠纷
```

**验收标准**:
- [ ] 申请售后（衣物损坏、描述不符、未收到等）
- [ ] 上传证据照片
- [ ] 平台客服介入处理
- [ ] 处理结果通知和押金调整

## 6. 用户旅程与流程设计

### 6.1 出租方流程
```
注册登录 → 实名认证 → 发布衣物 → 接收订单
   ↓          ↓          ↓          ↓
填写信息 → 上传身份证 → 填写详情 → 查看申请
   ↓          ↓          ↓          ↓
完成认证   系统审核    设置价格   确认/拒绝
                         ↓          ↓
                    上传照片    同意发货
                         ↓          ↓
                    发布成功   填写物流
                                   ↓
                              用户收货
                                   ↓
                              用户归还
                                   ↓
                              确认收到
                                   ↓
                              退回押金
```

### 6.2 租赁方流程
```
注册登录 → 浏览衣物 → 查看详情 → 下单租赁
   ↓          ↓          ↓          ↓
填写信息 → 筛选搜索    确认信息   选择租期
   ↓          ↓          ↓          ↓
完成认证   查看评分    联系出租方   确认订单
                                   ↓
                              支付押金租金
                                   ↓
                              等待发货
                                   ↓
                              确认收货
                                   ↓
                              使用衣物
                                   ↓
                              归还衣物
                                   ↓
                              等待退款
                                   ↓
                              评价体验
```

## 7. 非功能需求

### 7.1 性能要求
- 页面加载时间 < 3秒
- 图片上传时间 < 10秒/张（高清图）
- 支持并发用户数 > 1000

### 7.2 安全要求
- 用户数据加密存储
- 支付接口集成第三方安全支付
- 实名认证信息保护
- 防止刷单和恶意行为

### 7.3 可用性要求
- 系统可用性 > 99.5%
- 移动端适配（优先支持小程序）
- 界面简洁易用，符合用户习惯

## 11. 风险与应对

### 11.1 衣物损坏风险
**风险**：租赁方损坏衣物但拒绝赔偿
**应对**：
- 强制押金制度
- 出租方上传衣物状态照片
- 争议处理机制

### 11.2 卫生安全问题
**风险**：用户担心衣物不干净
**应对**：
- 要求出租方承诺清洗后再发货
- 提供"未清洗"投诉渠道
- 第二阶段引入平台清洗服务

### 11.3 用户信任问题
**风险**：用户不敢出租或租借
**应对**：
- 实名认证门槛
- 信用体系
- 平台担保交易
- 完善的争议处理
